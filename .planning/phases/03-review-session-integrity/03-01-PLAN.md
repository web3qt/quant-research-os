<objective>
Make `review_session` / local manual recovery visibly non-equivalent to normal independent `spawned_agent` review by blocking promotable PASS closure unless an explicit manual recovery contract is present, and by recording review execution governance in closure artifacts.
</objective>

<threat_model>
**Assets:** independent adversarial review guarantee, reviewer receipt proof chain, closure artifacts, downstream stage promotion.

**Threats:**
- **High:** A `local-review-session-*` or `review_session` receipt can currently write `CLOSURE_READY_PASS` and produce ordinary PASS closure without independent reviewer spawn.
- **High:** Downstream readers must inspect nested receipt/result payloads to distinguish spawned reviewer closure from manual recovery.
- **Medium:** Docs and skill surfaces can keep implying `qros-start-review` / `review_session` is an ordinary review path.

**Mitigations in this plan:**
- Add focused RED tests for `review_session` PASS rejection and explicit manual recovery allow-list behavior.
- Add deterministic runtime governance around `receipt_payload["execution_mode"]` and `review_loop_outcome`.
- Add a validated `review/request/manual_recovery_contract.yaml` contract for exceptional recovery.
- Record `review_execution_governance` in closure artifacts and trace events.
- Update shared protocol, core skills, user docs, and generated review skill template.
</threat_model>
