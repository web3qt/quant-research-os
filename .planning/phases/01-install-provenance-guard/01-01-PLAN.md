<objective>
Extend the QROS install manifest so every repo-local or global installation records enough provenance to diagnose source repository, commit, dirty-state, and Python interpreter drift before a consuming research repo trusts installed runtime assets.
</objective>

<threat_model>
**Assets:** Installed `.qros/install-manifest.json`, global `~/.codex/qros/install-manifest.json`, active research repo runtime wrappers, QROS source checkout.

**Threats:**
- **Medium:** A consuming research repo silently points to a stale or different QROS source checkout and produces review/session results from the wrong contracts.
- **Medium:** A dirty QROS source checkout is installed, later cleaned or changed, and wrappers continue without showing the provenance gap.
- **Low:** Manifest Python metadata is mistaken for execution enforcement before Phase 5 implements wrapper interpreter preference.

**Mitigations in this plan:**
- Record `source_repo_path`, `source_git_commit`, `source_git_dirty`, `source_git_status_short`, `python_executable`, and `python_version` in the manifest.
- Extend `check_install()` to compare installed provenance against current source checkout state and emit actionable diagnostics.
- Document that Phase 1 records interpreter provenance only; Phase 5 enforces wrapper interpreter selection.
</threat_model>
