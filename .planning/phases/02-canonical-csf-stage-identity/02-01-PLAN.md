<objective>
Make CSF stage identity canonical as `csf_*` everywhere the runtime, scaffold, and review/preflight machinery reason about stage identity, while preserving established lineage-local CSF program directories such as `program/cross_sectional_factor/data_ready`.
</objective>

<threat_model>
**Assets:** CSF stage gate identity, stage program manifests, lineage-local program directories, review/preflight validation, anti-drift snapshots.

**Threats:**
- **High:** Runtime session specs keep downgrading `csf_data_ready` to `data_ready`, so review/preflight checks see a different contract identity than the session flow.
- **High:** Canonical CSF ids are passed into program runtime and resolve to `program/cross_sectional_factor/csf_data_ready`, missing the existing lineage-local `data_ready` program.
- **Medium:** Scaffold-generated `stage_program.yaml` records legacy base ids for CSF stages, weakening provenance and review-entry identity checks.
- **Medium:** Tests may pass by asserting only key names or paths, while missing canonical `stage_id` drift.

**Mitigations in this plan:**
- Lock CSF spec identities in both `SESSION_STAGE_PROGRAM_SPECS` and `STAGE_PROGRAM_SPECS`.
- Add an explicit CSF canonical-id to local-program-dir mapping in `stage_program_relative_dir()`.
- Add focused tests that fail against the current legacy `csf_* -> base_stage` identity drift.
- Keep TSS and legacy time-series path behavior unchanged.
</threat_model>
