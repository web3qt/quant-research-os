<objective>
Make review request, canonical result, and closure artifacts runtime-owned and digest-validated, while blocking promotable PASS from unapproved `local-review-session-*` recovery lanes.
</objective>

<threat_model>
**Assets:** review request scope, reviewer receipt, raw reviewer findings, canonical review result, closure artifacts, stage evaluator progress decision, research-session current stage.

**Threats:**
- **High:** Launcher lane hand-writes `adversarial_review_result.yaml` and bypasses independent reviewer raw findings.
- **High:** `local-review-session-*` recovery mode produces `PASS` and is mistaken for normal independent review.
- **High:** Hand-written `review/closure/stage_completion_certificate.yaml` allows downstream progression even though closure was not produced by `qros-review`.
- **Medium:** Request or receipt fields are edited after `qros-review-cycle prepare`, weakening handoff scope while preserving enough fields to pass current validation.
- **Medium:** Tests keep using direct YAML fixtures that normalize unsafe artifact-writing patterns.

**Mitigations in this plan:**
- Add ownership metadata and canonical payload digest validation for request, result, and closure artifacts.
- Require explicit manual recovery metadata for review-session recovery PASS promotion.
- Require `qros-review` to regenerate canonical result from raw findings unless an existing canonical result proves runtime ownership for the active cycle.
- Make research session and stage evaluator reject invalid closure ownership before allowing progression.
- Update docs and tests so direct hand-written review artifacts are no longer presented as normal workflow.
</threat_model>
