# QROS 测试指南

## 作用域

本文件约束 `tests/` 子树。

## 目的

这里是真实生效的验证层，不是 `harness/tests/` 的演示副本。

## 测试规则

- 如果新增用户可见字段，应补存在性、字段映射或契约测试
- 如果新增 artifact、入口文档或治理入口，应补最小存在性测试
- 如果文档承担用户入口作用，应补入口链接、关键命令或存在性测试
- 测试名称应能反映它锁定的是哪一类契约或行为
- 优先新增独立、低耦合的测试文件，避免把不相关契约堆到同一个大测试里

## 不要做的事

- 不要只测 happy path 而完全不锁定文档/契约存在
- 不要让测试名和它真正保护的对象脱节
- 不要把测试写成依赖聊天上下文才能理解
