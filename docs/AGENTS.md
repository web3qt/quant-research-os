# QROS 文档指南

## 作用域

本文件约束 `docs/` 子树。

## 目的

这里是真实生效的解释层文档目录，不是 `harness/docs/` 的演示副本。

## 文档规则

- 文档优先解释“为什么存在”，其次才是“怎么用”
- 如果文档解释字段，优先做字段一一对应，而不是只写 stage-level 概述
- 如果文档引用某个 runtime 字段、artifact 或命令，名称应与实际产物一致
- 用户入口文档必须与当前 runtime 行为和测试夹具保持一致
- 如果新增或重命名文档中出现的 artifact、字段或命令，必须同步更新相关测试

## 不要做的事

- 不要复制整仓地图信息到每篇文档里
- 不要让字段说明与磁盘产物名称脱节
- 不要保留与当前 contracts、runtime 或 active skills 冲突的旧口径
