# Thin Generated Review Skills Design

## Purpose

QROS review skills currently mix two responsibilities:

- Agent workflow discipline: stage-entry guard, reviewer isolation, handoff, closure, failure routing.
- Stage truth: required outputs, formal gates, review checklist, rollback rules, downstream permissions.

This creates drift risk because stage truth also lives in `contracts/`. It also increases context load because every generated review skill inlines large stage-specific blocks before the agent reaches the current review cycle.

This design changes review skills into thin generated entrypoints. Stage-specific review truth is generated per review cycle from contracts into `review/request/`, then bound to the active handoff.

## Scope

In scope:

- Generated review skills under `skills/*/qros-*-review/SKILL.md`.
- `templates/skills/review-stage/SKILL.md.tmpl`.
- `runtime/scripts/gen_stage_review_skills.py`.
- `runtime/tools/review_skillgen/*` rendering and prepare support.
- `qros-review-cycle prepare` handoff/request generation.
- Review closure freshness checks for generated contract context.
- Tests proving thin skill generation, context rendering, prepare output, and stale context rejection.

Out of scope:

- Author skill generation.
- Replacing stage-specific review skill names with one generic review skill.
- Changing stage gate semantics.
- Changing artifact schemas.
- Changing normal user entrypoint behavior for `qros-research-session`.

## Design Principles

1. Contracts remain the truth source.
2. Review skills enforce behavior, not stage facts.
3. Review cycle context is generated on demand from current contracts and current author outputs.
4. Old context cannot prove new author outputs or changed contracts.
5. Reviewers read frozen handoff inputs and write only review result files.

## Proposed Architecture

The system will produce two kinds of review artifacts.

Static generated skill:

```text
skills/<stage_group>/<skill_name>/SKILL.md
```

The static skill contains:

- Stage identity and purpose.
- `qros-check-stage-entry --stage <stage_id> --lane review`.
- Shared review protocol pointer.
- Reviewer isolation and write-scope rules.
- Requirement to run `./.qros/bin/qros-review-cycle prepare`.
- Requirement to hand off runtime-generated context files.
- Verdict routing rules.

The static skill no longer inlines full stage truth blocks such as `required_outputs`, `formal_gate`, `review_checklist`, `rollback_rules`, or `downstream_permissions`.

Dynamic per-cycle context:

```text
<stage_dir>/review/request/stage_contract_context.yaml
<stage_dir>/review/request/stage_contract_context.md
```

The dynamic context is generated by `qros-review-cycle prepare`. It is part of the active review request and is frozen to the active `review_cycle_id`.

## Stage Contract Context Shape

`stage_contract_context.yaml` is the machine-readable context.

```yaml

stage_id: csf_data_ready
stage_name: CSF Data Ready
review_cycle_id: csf_data_ready-cycle-1
contract_sources:
  workflow_stage_gate: contracts/stages/workflow_stage_gates.yaml
  review_checklist: contracts/review/review_checklist_master.yaml
  artifact_contract: contracts/artifacts/csf_data_ready_artifacts.yaml
contract_digests:
  contracts/stages/workflow_stage_gates.yaml: 2c26b46b68ffc68ff99b453c1d30413413422d706483bfa0f98a5e886266e7ae

  contracts/artifacts/csf_data_ready_artifacts.yaml: baa5a0964d3320fbc0c6a922140453c8513ea24ab8fd0577034804a967248096
author_materialization_digest: 21f58d27f827d295ffcd860c65045685e3baf1ad4506caa0140113b316647534
required_inputs: []
required_outputs: []

  pass_all_of: []
  fail_any_of: []
review_checks:
  blocking: []
  reservation: []
  info: []
audit_only: []
rollback_rules: {}
downstream_permissions: {}

  required: true
  artifact_contract_code: ARTIFACT-CONTRACT-001
  semantic_code: CSF-DATA-SEMANTIC-001

  - Review current stage formal package credibility.

```

`stage_contract_context.md` is the reviewer-facing summary. It should be concise and should point to source contracts rather than copying them as a second truth source. It may include compressed bullet summaries of formal gate and review focus, but its source must be `stage_contract_context.yaml`.

## Data Flow

1. User or orchestrator enters a generated review skill.
2. Skill runs `./.qros/bin/qros-check-stage-entry --stage <stage_id> --lane review`.
3. Main thread completes review-ready self-check.
4. Main thread spawns an isolated reviewer.
5. Main thread runs `./.qros/bin/qros-review-cycle prepare ...`.
6. Prepare reads `contracts/stages/workflow_stage_gates.yaml`, `contracts/review/review_checklist_master.yaml`, and `contracts/artifacts/<stage>_artifacts.yaml`.
7. Prepare computes contract digests and current author materialization digest.
8. Prepare writes `review/request/stage_contract_context.yaml` and `review/request/stage_contract_context.md`.
9. Prepare writes handoff/request files that explicitly reference the stage contract context files.
10. Reviewer reads only `review/request/*` and `author/formal/*`.
11. Reviewer writes only the result file declared by the active shared review protocol and the active prepare output. The context renderer must not invent a separate reviewer output path.
12. Main thread runs `./.qros/bin/qros-review`.
13. Closure verifies review cycle identity, author digest, and contract context digest before accepting the result.

## Error Handling

Prepare must fail closed if it cannot generate valid stage contract context.

Required error conditions:

- Missing workflow stage gate entry.
- Missing review checklist entry.
- Missing artifact contract for stages that require artifact contract validation.
- Invalid or non-serializable context payload.
- Missing author materialization digest when the review request requires author output binding.

Suggested reason codes:

- `REVIEW_CONTRACT_CONTEXT_MISSING`
- `REVIEW_CONTRACT_CONTEXT_INVALID`
- `REVIEW_CONTRACT_CONTEXT_STALE`

No fallback to embedded skill gate text is allowed.

## Drift Protection

The generated context must bind both contracts and author outputs.

Closure must reject the review if:

- Current contract digests differ from the digests recorded by prepare.
- Current author materialization digest differs from the digest recorded by prepare.
- Reviewer result references a different `review_cycle_id`.
- `review/request/stage_contract_context.*` changed after prepare.

Recovery action is explicit: rerun `qros-review-cycle prepare` and start a fresh review handoff.

## Reviewer Write Scope

Allowed read roots:

```text
review/request/*
author/formal/*
```

Allowed write roots:

```text
review/result/*
```

Disallowed writes:

```text
review/request/stage_contract_context.*
author/formal/*
contracts/*
```

Existing reviewer write-scope audit should continue to enforce this boundary. The new context files are launcher/runtime-owned request artifacts, not reviewer-owned output artifacts.

## Generated Skill Template Changes

`templates/skills/review-stage/SKILL.md.tmpl` should remove full inline blocks for:

- `{{REQUIRED_INPUTS_BLOCK}}`
- `{{REQUIRED_OUTPUTS_BLOCK}}`
- `{{FORMAL_GATE_BLOCK}}`
- `{{CHECKLIST_BLOCK}}`
- `{{AUDIT_ONLY_BLOCK}}`
- `{{ROLLBACK_BLOCK}}`
- `{{DOWNSTREAM_BLOCK}}`

The replacement should be a short section:

```md
## Stage Contract Context

Do not reconstruct stage gates from memory or from this skill.
Run `qros-review-cycle prepare` and use the generated files:

- `review/request/stage_contract_context.yaml`
- `review/request/stage_contract_context.md`

These files are the review-cycle-local rendering of current contracts and current author outputs.
```

The renderer may keep a short stage name and purpose in the skill front matter/body for discoverability.

## Tests

Add or update tests in four groups.

Context renderer tests:

- Stage id resolves to the expected artifact contract path.
- Workflow gate, review checklist, and artifact contract render into YAML.
- Markdown context includes source paths and concise stage summary.
- Context includes contract digests and author materialization digest.
- Missing contract input fails closed.

Prepare integration tests:

- `qros-review-cycle prepare` writes `stage_contract_context.yaml`.
- `qros-review-cycle prepare` writes `stage_contract_context.md`.
- Handoff manifest or prompt references both context files.
- Context `review_cycle_id` matches active request.

Thin skill generation tests:

- Generated review skill includes `qros-check-stage-entry`.
- Generated review skill includes `qros-review-cycle prepare`.
- Generated review skill references `stage_contract_context.yaml` and `stage_contract_context.md`.
- Generated review skill does not inline full formal gate/checklist/required output blocks.
- Existing freshness test still detects drift in generated skill files.

Stale context closure tests:

- Prepare then modify `author/formal/*`; closure rejects review.
- Prepare then change a contract digest; closure rejects review.
- Prepare then modify `review/request/stage_contract_context.*`; closure rejects review.
- Rerunning prepare produces a fresh context that can be reviewed.

## Migration Plan

Phase 1: Add context renderer.

- Create renderer helpers in `runtime/tools/review_skillgen/`.
- Render YAML and Markdown context from existing contract loaders.
- Add unit tests.

Phase 2: Wire prepare.

- Update review cycle prepare to write context files.
- Add context paths to handoff/request output.
- Add integration tests.

Phase 3: Thin generated review skills.

- Shrink `templates/skills/review-stage/SKILL.md.tmpl`.
- Update `runtime/tools/review_skillgen/render.py`.
- Regenerate Codex and Claude Code review skills.
- Add thin skill tests and keep freshness tests.

Phase 4: Closure freshness.

- Persist context digests in active review request or runtime state.
- Verify context, contracts, author digest, and review cycle id during closure.
- Add stale context tests.

Phase 5: Documentation.

- Update review usage docs to say generated review skills are workflow entrypoints, not stage truth.
- Document `stage_contract_context.yaml` and `.md`.

## Success Criteria

- Generated review skills are materially shorter.
- Stage-specific truth is no longer duplicated in generated review skill bodies.
- Each review cycle has frozen `stage_contract_context.yaml` and `.md`.
- Closure rejects reviews bound to stale contracts or stale author outputs.
- Existing public review skill names remain available.
- No author skill behavior changes.

## Non-Goals

- This does not change QROS stage semantics.
- This does not remove stage-specific review skills.
- This does not redesign `qros-research-session`.
- This does not implement author skill generation.
